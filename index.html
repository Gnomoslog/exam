1. История создания языка Python. Сферы применения

Python разработан Гвидо ван Россумом в 1991 году как язык с акцентом на читаемость кода и производительность разработчика. Название вдохновлено комедийным шоу «Монти Пайтон».

📌 Примеры применения:
  • Веб-разработка: Instagram (Django), YouTube (частично) — для обработки миллионов запросов
  • Научные вычисления: NASA использует Python для расчётов траекторий
  • Искусственный интеллект: ChatGPT, рекомендательные системы Netflix
  • Финансовый сектор: JPMorgan использует Python для анализа рынков
  • Автоматизация: Сотни компаний автоматизируют рутинные задачи (отчёты, обработка почты)
2. Термин API. Возможности применения API

API (Application Programming Interface) — это интерфейс, который позволяет разным программам обмениваться данными по чётко определённым правилам. По сути, это посредник между сервисами.

📌 Реальные примеры использования:
  • Погода в приложении: Ваше приложение отправляет запрос к API OpenWeatherMap и получает данные о погоде
  • Оплата через интернет: Интернет-магазин подключается к API платёжной системы Stripe для проведения транзакций
  • Карты и навигация: Приложения используют Google Maps API для отображения карт
  • Социальные сети: Посты в Instagram автоматически публикуются через API
  • Банковские операции: Мобильные банки используют API для получения баланса

  • CI/CD: В системах непрерывной интеграции аргументы передают версию сборки и окружение (dev/stage/prod)

1. Циклические алгоритмы

Циклы — это когда одно и то же действие повторяется несколько раз. for — для перебора элементов, while — пока условие верно.

📌 Примеры:
for i in range(5):
print(i) — напечатает 0, 1, 2, 3, 4

x = 0
while x < 5:
print(x)
x = x + 1 — напечатает 0, 1, 2, 3, 4

1. Объекты – кортеж, множества

Кортеж — это как список, но его нельзя изменить (замороженный список). Множество — это как мешок, в котором все предметы уникальные.

📌 Примеры:

s.union({3, 4, 5}) — объединение множеств

2. Kanban методология

Kanban — это доска с колонками: «Сделать», «В работе», «Готово». Задачи переезжают из одной колонки в другую.

📌 Пример доски Kanban:
Сделать (10 задач) → В работе (3 задачи) → Готово (5 задач)

Правила:
• Ограничивать количество задач в работе (WIP) — например, не больше 3
• Задачи переезжают слева направо
• Нельзя брать новую задачу, если в работе уже есть лимит
• Визуализация всего процесса помогает видеть узкие места
