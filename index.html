1. Использование командной строки Python Shell. Основные принципы работы с аргументами командной строки
Python Shell (интерактивный режим, REPL — Read-Eval-Print Loop) — это среда, где можно писать и выполнять код построчно, сразу видя результат. Запускается командой python в терминале. Позволяет тестировать небольшие фрагменты кода, изучать библиотеки, отлаживать. Аргументы командной строки — это параметры, которые передаются программе при запуске через пробел. Основные принципы: аргументы передаются в виде строк, их нужно преобразовывать к нужным типам, программа должна корректно обрабатывать отсутствие обязательных аргументов.
📌 Примеры:
  • python script.py data.txt — передача одного аргумента
  • python script.py --file data.csv --mode analyze — именованные аргументы
  • sys.argv[1] — получение первого аргумента в коде
  • argparse.ArgumentParser() — создание парсера для сложных параметров

1. Работа с переменными в языке Python
Переменная — это именованная область памяти, которая хранит значение. В Python не нужно объявлять тип переменной — он определяется автоматически при присваивании (динамическая типизация). Переменная создаётся в момент первого присваивания. Имена переменных должны начинаться с буквы или подчёркивания, могут содержать буквы, цифры и подчёркивания, чувствительны к регистру. Нельзя использовать зарезервированные слова (if, for, while и др.). Переменные могут менять тип в процессе выполнения программы (например, сначала число, потом строка). Для удаления переменной используется del. Для обмена значениями достаточно a, b = b, a.
📌 Примеры:
age = 25 — целое число
name = "Мария" — строка
items = ["хлеб", "молоко"] — список
score = score + 10 — увеличение значения
a, b = 5, 10 — множественное присваивание
del x — удаление переменной

1. Условные конструкторы
Условные конструкции (условные операторы) позволяют выполнять разные блоки кода в зависимости от условий. Основная конструкция: if — проверяет условие, если оно истинно, выполняет блок; elif — проверяет дополнительное условие (может быть несколько); else — выполняется, если все предыдущие условия ложны. Условия могут быть составными с использованием логических операторов and, or, not. Также существует тернарный оператор (условное выражение): x = a if condition else b — сокращённая форма. Условные конструкции могут быть вложенными (if внутри if). Важно соблюдать отступы (indentation) — в Python они определяют блоки кода.
📌 Примеры: if age >= 18:
print("Взрослый")
elif age >= 14:
print("Подросток")
else:
print("Ребёнок")

Тернарный оператор:
status = "Взрослый" if age >= 18 else "Ребёнок"

1. Циклические алгоритмы
Циклические алгоритмы позволяют многократно выполнять один и тот же блок кода. В Python есть два основных типа циклов: for и while. Цикл for используется для итерации по последовательности (список, строка, словарь, range): for item in collection:. Цикл while выполняется, пока условие истинно: while condition:. Управление циклом: break — полный выход из цикла; continue — переход к следующей итерации; else — выполняется, если цикл завершился нормально (без break). Циклы могут быть вложенными. Важно избегать бесконечных циклов (при использовании while нужно менять условие внутри). Для оптимизации используют генераторы и списковые включения.
📌 Примеры: for i in range(5):
print(i) — напечатает 0, 1, 2, 3, 4

x = 0
while x < 5:
print(x)
x += 1 — напечатает 0, 1, 2, 3, 4

for i in range(10):
if i == 5: break — остановится на 5

2. Дерево виджетов – как основа пользовательского интерфейса
В Kivy интерфейс строится как дерево виджетов — иерархическая структура, где корневой виджет (Root Widget) содержит дочерние виджеты, а те, в свою очередь, могут содержать свои дочерние элементы. Доступ к дереву осуществляется через свойство children (список дочерних виджетов) и методы поиска по id. Правильная организация дерева важна для производительности и удобства разработки.

self.ids.my_widget — поиск по id

1. Объекты – кортеж, множества
Кортеж (tuple) — упорядоченная неизменяемая коллекция. Создаётся круглыми скобками: (1, 2, 3). Нельзя изменять, добавлять или удалять элементы после создания. Используется для защиты данных от изменений, как ключи в словарях, для возврата нескольких значений из функции. Множество (set) — неупорядоченная коллекция уникальных элементов. Создаётся фигурными скобками: {1, 2, 3}. Не может содержать дубликаты (автоматически удаляются). Методы: add() — добавить элемент; remove() — удалить (с ошибкой, если нет); discard() — удалить (без ошибки); union() — объединение множеств; intersection() — пересечение; difference() — разность; symmetric_difference() — симметрическая разность; isdisjoint() — проверка на отсутствие общих элементов; issubset()/issuperset() — проверка вхождения. Множества полезны для удаления дубликатов и выполнения математических операций над коллекциями.
📌 Примеры:
t = (1, 2, 3) — кортеж (неизменяемый)
a, b = (10, 20) — распаковка кортежа

s = {1, 2, 3, 3} → {1, 2, 3} (дубликат удалён)
s.add(4) → {1, 2, 3, 4}
s.union({3, 4, 5}) → {1, 2, 3, 4, 5}
2. Виджеты для позиционирования элементов интерфейса в приложениях на Kivy
В Kivy есть несколько виджетов-контейнеров для позиционирования дочерних элементов. FloatLayout — позволяет задавать точные координаты и размеры (pos и size), дочерние виджеты могут перекрываться. BoxLayout — выстраивает элементы в ряд (horizontal) или колонку (vertical), автоматически распределяя пространство. GridLayout — размещает элементы в сетке с заданным количеством колонок (cols) или строк (rows). Каждый виджет имеет свои свойства для настройки.
📌 Примеры:

1. Передача аргументов в параметры функций. Лямбда функции в языке Python
Аргументы в функции могут передаваться разными способами. Позиционные аргументы — передаются в том порядке, в котором объявлены параметры. Ключевые аргументы (по имени) — передаются с указанием имени параметра (например, func(a=5, b=10)). Значения по умолчанию задаются при объявлении функции (def func(a, b=10):). Аргументы могут быть обязательными и необязательными. *args — собирает все позиционные аргументы в кортеж. **kwargs — собирает все ключевые аргументы в словарь. Лямбда-функции (lambda) — анонимные функции, создаваемые в одну строку: lambda x, y: x + y. Они не имеют имени, могут содержать только одно выражение (возвращаемое значение). Используются как аргументы функций высшего порядка (map, filter, sorted) и для коротких простых операций.
📌 Примеры:
def func(a, b=10): return a + b — значение по умолчанию
func(5, b=20) — ключевой аргумент
def func(*args, **kwargs) — переменное число аргументов

double = lambda x: x * 2

1. Анонимные функции и их применение внутри функций высшего порядка
Анонимные функции (лямбда-функции) — это функции без имени, создаваемые с помощью ключевого слова lambda. Они могут принимать любое количество аргументов, но содержат только одно выражение, которое автоматически возвращается. Лямбда-функции часто используются внутри функций высшего порядка — функций, которые принимают другие функции в качестве аргументов или возвращают их. Основные функции высшего порядка в Python: map() — применяет функцию ко всем элементам коллекции; filter() — фильтрует элементы по условию; sorted() — сортировка с ключом; reduce() — свёртка коллекции (из модуля functools); max(), min() — поиск максимума/минимума по ключу. Лямбды делают код лаконичнее, когда функция нужна только один раз и не требует отдельного объявления.
📌 Примеры:
list(map(lambda x: x**2, [1,2,3])) → [1, 4, 9]
list(filter(lambda x: x % 2 == 0, [1,2,3,4])) → [2, 4]
sorted(people, key=lambda person: person['age']) — сортировка по возрасту
max(data, key=lambda x: x['score']) — максимум по оценке

В Kivy доступ к виджетам в дереве осуществляется несколькими способами. По индексу: root.children[0] — обращение к первому дочернему элементу (порядок обратный — последний добавленный имеет индекс 0). 📌 Примеры:
root.children[0] — первый дочерний виджет
self.ids.username.text — получить текст из поля с id="username"

for child in root.walk(): — обход всех виджетов

1. Работа синхронных функций. Создание асинхронной функции
Синхронные функции — это обычные функции, которые выполняются последовательно: каждая следующая операция ждёт завершения предыдущей. Это может приводить к блокировке (например, при загрузке данных из интернета программа «зависает»). Асинхронные функции (создаются с помощью async def) позволяют выполнять операции без блокировки. Внутри асинхронной функции используется await для вызова других асинхронных функций (ожидание, но без блокировки всего потока). Асинхронность позволяет эффективно использовать ресурсы при вводе-выводе (сеть, файлы, базы данных). Для выполнения асинхронных функций используется asyncio.run() или event loop. Асинхронные функции не работают с синхронным кодом без специальных обёрток. Правильное использование асинхронности повышает производительность сетевых приложений.
📌 Примеры:
Синхронная:
data = load_file() — ждём загрузки
process_data(data) — потом обрабатываем

Асинхронная:
async def fetch_data():
await asyncio.sleep(1)
return "данные"
result = await fetch_data()

Рекомендации: начинать с начального узла и заканчивать конечным; избегать слишком детализированных диаграмм; использовать понятные названия действий; показывать альтернативные пути через ветвления; группировать связанные действия в зоны ответственности (swimlanes); проверять полноту всех возможных путей; диаграмма должна быть читаема и понятна всем участникам.

2. Agile методология
Agile — это набор принципов и подходов к разработке программного обеспечения, основанный на гибком реагировании на изменения. Манифест Agile был создан в 2001 году и содержит 12 принципов. 1. Требования к интерфейсу пользователя. Принципы создания графического пользовательского интерфейса (GUI)
Требования к интерфейсу пользователя делятся на функциональные (что должен делать интерфейс) и нефункциональные (как должен работать). Основные нефункциональные требования: удобство использования (usability) — интуитивность и лёгкость обучения; доступность (accessibility) — поддержка людей с ограниченными возможностями; производительность — быстрая реакция на действия; эстетичность — визуальная привлекательность; согласованность — единые стандарты во всем приложении; отказоустойчивость — работа при ошибках. Принципы создания GUI: простота — минимум действий для достижения цели; обратная связь — пользователь знает результат своих действий; понятная навигация — всегда понятно, где находишься и куда можно пойти; предотвращение ошибок — блокировка неверных действий; восстановление после ошибок — возможность отменить действие; F-образный макет (для чтения) и правило 3-х кликов — максимум 3 клика до нужного действия; использование стандартных паттернов (кнопки, меню, иконки).
📌 Примеры принципов:
• Кнопка "Купить" всегда на видном месте, яркая
• При нажатии появляется анимация или сообщение об успехе
• Все страницы имеют "Назад" или хлебные крошки
• Невозможно отправить пустую форму — кнопка неактивна
• Есть возможность отменить последнее действие
2. Scrum методология
Scrum — это фреймворк для управления сложными проектами, основанный на принципах Agile. Ключевые элементы: спринт — временной отрезок (обычно 2-4 недели), в течение которого создаётся готовый к поставке инкремент продукта. Роли: Product Owner — определяет функциональность продукта и приоритеты (формирует бэклог); Scrum Master — фасилитатор, помогает команде соблюдать Scrum-процессы, устраняет препятствия; Команда разработки — кросс-функциональная группа (5-9 человек), которая выполняет работу. Артефакты: Бэклог продукта — список всех требований; Бэклог спринта — задачи, выбранные на спринт; Инкремент — работающий продукт после спринта. События: Sprint Planning — планирование спринта; Daily Scrum — ежедневная 15-минутная планерка; Sprint Review — демонстрация результата заказчику; Sprint Retrospective — анализ процесса и поиск улучшений. Scrum подходит для проектов с быстро меняющимися требованиями.
📌 Пример работы Scrum:
Команда из 6 человек работает 2 недели над новыми функциями. Каждый день в 10:00 проводят 15-минутную планерку (что сделано, что планируется, какие проблемы). Через 2 недели показывают заказчику готовый инкремент. Затем обсуждают, что можно улучшить в процессе, и планируют следующий спринт.

1. Понятие спецификации языка программирования. Синтаксис языка программирования. Стиль программирования
Спецификация языка программирования — это официальный документ, который описывает все аспекты языка: синтаксис, семантику, стандартную библиотеку, поведение интерпретатора/компилятора. Спецификация определяет, что является корректной программой на данном языке, как она должна выполняться и какие результаты выдавать. Синтаксис — это набор правил, определяющих, как должны писаться конструкции языка (переменные, операторы, функции, классы). Синтаксис включает правила расстановки знаков препинания, скобок, отступов. Стиль программирования — это неформальные рекомендации и соглашения по оформлению кода, которые делают его читаемым и поддерживаемым. В Python стиль определён в PEP 8: отступы (4 пробела), максимальная длина строки (79 символов), именование (snake_case для переменных и функций, CamelCase для классов), пробелы вокруг операторов, комментарии. Соблюдение стиля упрощает совместную работу и снижает количество ошибок.
📌 Примеры правил:
Синтаксис: после if нужно ставить двоеточие :, а блок кода выделять отступом
Стиль: def calculate_average(scores): — имя функции в нижнем регистре с подчёркиваниями
PEP 8: не более 79 символов в строке, 4 пробела для отступа
2. Lean методология
Lean (бережливое производство) — это методология управления, направленная на устранение всех видов потерь и оптимизацию процессов. В разработке ПО Lean применяется для повышения эффективности и сокращения времени от идеи до реализации. Основные принципы: ценность — определять ценность с точки зрения заказчика; карта потока создания ценности — визуализировать все шаги процесса; поток — обеспечивать непрерывное движение ценности; вытягивание — делать только то, что действительно нужно; совершенство — непрерывно улучшать процессы. Lean-практики: сокращение избыточной документации; минимизация времени между написанием кода и его развёртыванием; быстрая обратная связь; устранение узких мест; автоматизация рутинных задач; постоянное обучение команды. Lean часто комбинируется с Agile и Kanban. Цель Lean — делать больше с меньшими затратами.
📌 Примеры Lean-практик:
• Вместо 100-страничной документации — краткое описание на 5 страниц
• Вместо функций, которые никто не использует — только то, что нужно сейчас
• Каждую неделю спрашивать пользователей, что им нужно
• Автоматизация сборки и развёртывания (CI/CD)

2. Kanban методология
Kanban — это метод управления потоком работы, визуализируемый на доске с колонками, отражающими стадии процесса. Основные колонки: «To Do» (Сделать) — задачи, которые нужно выполнить; «In Progress» (В работе) — задачи, над которыми идёт работа; «Done» (Готово) — завершённые задачи. Ключевые принципы Kanban: визуализация потока работы — чтобы все видели текущее состояние дел; ограничение незавершённой работы (WIP — Work In Progress) — нельзя брать больше определённого количества задач одновременно; управление потоком — оптимизация для сокращения времени выполнения; непрерывное улучшение — регулярный анализ и оптимизация процесса. Kanban не требует фиксированных итераций (спринтов), работа ведётся непрерывно. Приоритет задач может меняться в любой момент. Kanban подходит для проектов с постоянно поступающими задачами (поддержка, обслуживание). Важно: регулярно пересматривать WIP-лимиты и улучшать процесс.
📌 Пример доски Kanban:
Сделать (10 задач) → В работе (3 задачи, WIP=3) → Готово (5 задач)

Правила: программист берёт задачу из «Сделать» только если в «В работе» меньше 3 задач. Когда задача готова, она переезжает в «Готово». Каждое утро команда смотрит на доску и обсуждает узкие места.
