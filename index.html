📚 Ответы на экзаменационные билеты

Билет №1
1. История создания языка Python. Сферы применения

Создан Гвидо ван Россумом в 1991 году. Название — отсылка к комедийному шоу «Летающий цирк Монти Пайтона». Применяется в веб-разработке (Django, Flask), научных вычислениях (NumPy, SciPy), машинном обучении (TensorFlow, PyTorch), автоматизации, написании скриптов, разработке игр (Pygame) и GUI-приложений.

2. Термин API. Возможности применения API

API (Application Programming Interface) — это набор правил и инструментов для взаимодействия между программами. Позволяет интегрировать сервисы (Google Maps, платежные системы), получать данные (погода, курсы валют), автоматизировать задачи и расширять функциональность приложений.

Билет №2
1. Использование командной строки Python Shell. Основные принципы работы с аргументами командной строки

Python Shell (REPL) позволяет выполнять код в интерактивном режиме. Аргументы командной строки передаются через sys.argv, где sys.argv[0] — имя скрипта, а остальные — параметры. Используется модуль argparse для парсинга сложных аргументов.

2. Технология написания бота на языке Python

Ботов пишут с использованием библиотек: python-telegram-bot, aiogram (для Telegram), discord.py (для Discord). Основной принцип: обработка событий (команды, сообщения), использование API мессенджеров, асинхронность для высокой производительности.

Билет №3
1. Типы данных языка Python

Основные: int, float, complex (числа); str (строки); bool (логический); list, tuple, dict, set, frozenset (коллекции); NoneType. Также есть bytes, bytearray, memoryview и др.

2. Инструментальные средства для разработки кроссплатформенных приложений на Python

Основные фреймворки: Kivy (для мобильных и десктопных приложений), BeeWare (Toga), PyQt/PySide (для десктопа), Flask/Django (веб-приложения, доступны на всех платформах).

Билет №4
1. Работа с переменными в языке Python

Переменные создаются при первом присваивании (динамическая типизация). Имена чувствительны к регистру, могут содержать буквы, цифры и '_'. Нельзя использовать зарезервированные слова. Пример: x = 10, name = "John".

2. Фреймоворк Kivy, язык KV и виджеты, как основа пользовательского интерфейса

Kivy — открытый фреймворк для создания мультитач-приложений. Язык KV (Kivy Language) — декларативный язык для описания интерфейса. Виджеты — базовые элементы UI (Button, Label, TextInput и др.), которые строятся в дерево виджетов.

Билет №5
1. Встроенные функции языка Python

Примеры: print(), len(), type(), int(), str(), list(), dict(), sorted(), sum(), min(), max(), open(), input(), range(), enumerate(), zip(), map(), filter(), и многие другие.

2. Зарезервированные слова и выражения в языке KV

Ключевые слова в KV: canvas, on_press, on_release, id, pos, size, text, color, background_color, font_size, orientation, spacing, padding, size_hint, pos_hint и др. Используются для настройки свойств виджетов.

Билет №6
1. Основные операторы языка Python

Арифметические (+, -, *, /, //, %, **), сравнения (==, !=, >, <, >=, <=), логические (and, or, not), присваивания (=, +=, -= и др.), принадлежности (in, not in), тождественности (is, is not).

2. Компоновка приложения из фрагментов методом соглашения имен

В Kivy используется kv-файл с тем же именем, что и класс приложения (например, MyApp -> my.kv). Это позволяет автоматически связывать логику и интерфейс. Также используют id для обращения к виджетам из Python-кода.

Билет №7
1. Методы работы с типами данных: строка, число

Строки: lower(), upper(), strip(), split(), join(), replace(), find(), format(), f-строки. Числа: int(), float(), round(), abs(), pow(), math.floor(), math.ceil(), преобразования между системами счисления.

2. Виджеты в Kivy

Основные виджеты: Label (текст), Button (кнопка), TextInput (ввод текста), Image (изображение), Slider (ползунок), Switch (переключатель), CheckBox (флажок), ProgressBar (индикатор прогресса), DropDown (выпадающий список).

Билет №8
1. Использование операторов языка Python

Операторы используются для выполнения операций над данными. Например: a = 5 + 3 (арифметический), if x > 10: (сравнения), if a and b: (логический), for i in range(5): (циклический).

2. Задание размеров и положения виджетов в окне приложения Kivy

Используются свойства size (фиксированный размер), size_hint (доля от родителя), pos (фиксированная позиция), pos_hint (относительная позиция, например {'x': 0.5, 'y': 0.5}).

Билет №9
1. Условные конструкторы

if, elif, else. Также тернарный оператор: x = a if condition else b. Могут быть вложенными. Используются для ветвления логики программы.

2. Задание виджетам цвета фона

В Kivy используется свойство background_color или canvas.before с Color и Rectangle. В KV: background_color: 1, 0, 0, 1 (красный). Для Label используется canvas.before для заливки фона.

Билет №10
1. Циклические алгоритмы

Циклы for (итерация по коллекциям) и while (выполняется пока условие истинно). Управление: break (выход), continue (переход на следующую итерацию), else (выполняется после нормального завершения цикла).

2. Обработка событий виджетов

События в Kivy обрабатываются через привязку методов: on_press, on_release, on_click, on_text, on_slider_value и др. В KV: on_press: app.do_something(). В Python: btn.bind(on_press=self.callback).

Билет №11
1. Структуры данных языка Python: списки, словари. Методы работы со списками, словарями

Списки: append(), extend(), insert(), remove(), pop(), index(), count(), sort(), reverse(). Словари: keys(), values(), items(), get(), pop(), update(), setdefault(). Доступ по ключу.

2. Дерево виджетов – как основа пользовательского интерфейса

В Kivy UI строится как дерево виджетов, где корневой виджет (Root Widget) содержит дочерние. Дочерние виджеты могут иметь свои дочерние элементы. Управление иерархией через add_widget() и remove_widget().

Билет №12
1. Объекты – кортеж, множества

Кортеж (tuple) — неизменяемая упорядоченная коллекция (a = (1, 2, 3)). Множество (set) — неупорядоченная коллекция уникальных элементов (s = {1, 2, 3}). Методы: add(), remove(), union(), intersection(), difference().

2. Виджеты для позиционирования элементов интерфейса в приложениях на Kivy

FloatLayout (свободное позиционирование), BoxLayout (горизонтальное/вертикальное выравнивание), GridLayout (сетка), AnchorLayout (привязка к сторонам), RelativeLayout (относительное позиционирование).

Билет №13
1. Понятие функции. Конструкции и методы создания функций в языке Python

Функция — блок кода, который выполняется при вызове. Создаётся с помощью def имя(параметры):. Может возвращать значение через return. Пример: def add(a, b): return a + b.

2. Виджет Carousel для пролистывания слайдов

Carousel — виджет Kivy для создания галереи с пролистыванием (свайпом). Содержит слайды (дочерние виджеты). Настройка: loop (зацикливание), direction (направление), duration (время анимации).

Билет №14
1. Передача аргументов в параметры функций. Лямбда функции в языке Python

Аргументы передаются по позиции или по имени (ключевые аргументы). Можно задавать значения по умолчанию. Лямбда-функции — анонимные функции: lambda x, y: x + y. Используются в map(), filter(), sort() и др.

2. Задание цвета фона виджету – контейнеру

Для контейнеров (например, BoxLayout) цвет фона задаётся через canvas.before: Color и Rectangle. В KV: canvas.before: Color(rgba: 0.2, 0.2, 0.2, 1); Rectangle(pos: self.pos, size: self.size).

Билет №15
1. Анонимные функции и их применение внутри функций высшего порядка

Анонимные функции (lambda) часто используются как аргументы функций высшего порядка: map(lambda x: x**2, numbers), filter(lambda x: x > 0, numbers), sorted(data, key=lambda x: x['age']).

2. Индексация элементов в дереве виджетов

Доступ к дочерним виджетам через индексы: root.children[0] (последний добавленный). Для поиска по id: self.ids.my_widget. Также можно использовать recurrency для обхода дерева.

Билет №22
1. Понятие объекта. Создание методов объекта и их полей

Объект — экземпляр класса. Поля (атрибуты) и методы определяются в классе. Для создания объекта: obj = MyClass(). Методы определяются как def method(self):, обращение к полям через self.field.

2. Стили KivyMD для задания цвета элементам интерфейса

В KivyMD цвета задаются через тему (ThemeManager): primary_palette, primary_hue. Для отдельных элементов: md_bg_color, text_color, accent_color. Используются предопределённые цвета Material Design (Red, Blue, Green и др.).

Билет №23
1. Свойства ООП: понятие инкапсуляции, наследования, полиморфизма

Инкапсуляция — сокрытие внутреннего состояния (приватные атрибуты _ и __). Наследование — создание нового класса на основе существующего (class Child(Parent)). Полиморфизм — способность объектов разных классов отвечать на одни и те же методы по-своему.

2. Темы KivyMD для настройки цветовых оттенков

В KivyMD темы задаются через ThemeManager: theme_cls.primary_palette = "Blue", theme_cls.primary_hue = "500", theme_cls.theme_style = "Dark" или "Light". Также можно использовать accent_palette и accent_hue.

Билет №24
1. Работа магических методов.

Магические методы (__init__, __str__, __len__, __add__, __getitem__, __call__ и др.) позволяют переопределять поведение объектов. Вызываются автоматически при определённых операциях (создание объекта, сложение, вызов как функции и т.д.).

2. Компоненты KivyMD для позиционирования элементов интерфейса

MDBoxLayout, MDGridLayout, MDFloatLayout, MDAnchorLayout. Они работают аналогично стандартным Kivy, но с учётом Material Design. Также есть MDScreen как корневой экран для приложений.

Билет №25
1. Переопределение действий магических методов

Для переопределения нужно создать метод в классе. Например, __add__ для перегрузки оператора +, __eq__ для сравнения, __str__ для строкового представления. Это позволяет задать кастомное поведение объектов.

2. Компоненты пользовательского интерфейса KivyMD

Основные MD-виджеты: MDToolbar (верхняя панель), MDRectangleFlatButton, MDRaisedButton (кнопки), MDTextField (поле ввода), MDList (список), MDCard (карточка), MDDialog (диалоговое окно), MDNavigationDrawer (боковое меню).

Билет №26
1. Диаграммы деятельности. Рекомендации по построению диаграмм деятельности

Диаграммы деятельности (Activity Diagram) — UML-диаграммы для моделирования бизнес-процессов и алгоритмов. Рекомендации: начинать с начального узла, использовать ветвления (decision), параллельные потоки (fork/join), завершать конечным узлом.

2. Гибкие методологии проектирования

Гибкие методологии (Agile) ориентированы на итеративную разработку, частую поставку работающего ПО, тесное взаимодействие с заказчиком. Основные принципы: адаптивность, простота, командная работа, обратная связь.

Билет №27
1. Диаграммы последовательности. Рекомендации по построению диаграмм последовательности

Диаграмма последовательности (Sequence Diagram) показывает взаимодействие объектов во времени. Рекомендации: обозначать объекты и их линии жизни, использовать стрелки для сообщений, нумеровать шаги, показывать возвраты, не перегружать деталями.

2. Agile методология

Agile — это набор принципов и подходов к разработке ПО, основанный на итерациях, самоорганизации команд, частой поставке ценности заказчику. Включает Scrum, Kanban, Extreme Programming (XP) и др.

Билет №28
1. Требования к интерфейсу пользователя. Принципы создания графического пользовательского интерфейса (GUI)

Требования: удобство, интуитивность, эстетичность, отзывчивость, доступность. Принципы: согласованность, обратная связь, понятная навигация, минимализм, предотвращение ошибок, использование стандартных паттернов (F-образный макет, правило 3-х кликов).

2. Scrum методология

Scrum — гибкая методология с фиксированными спринтами (обычно 2-4 недели). Роли: Product Owner (формирует бэклог), Scrum Master (фасилитатор), Команда. Артефакты: бэклог спринта, инкремент продукта. События: Sprint Planning, Daily Stand-up, Sprint Review, Sprint Retrospective.

Билет №29
1. Понятие спецификации языка программирования. Синтаксис языка программирования. Стиль программирования

Спецификация — официальный документ, описывающий синтаксис, семантику и стандартную библиотеку. Синтаксис — правила написания конструкций (переменные, операторы, функции). Стиль (PEP 8) — рекомендации по оформлению кода (отступы, имена, пробелы).

2. Lean методология

Lean (бережливое производство) — методология, направленная на устранение потерь и оптимизацию процессов. В разработке ПО: минимизация ненужной документации, сокращение времени между идеей и реализацией, быстрая обратная связь, непрерывное улучшение.

Билет №30
1. Разработка графического интерфейса пользователя

Этапы: анализ требований, проектирование (wireframes, макеты), выбор фреймворка (Kivy, PyQt, Tkinter), разработка интерфейса (верстка, стилизация), реализация логики, тестирование (юзабилити, кроссплатформенность), развертывание.

2. Kanban методология

Kanban — метод управления потоком работы, визуализируемый на доске (столбцы: To Do, In Progress, Done). Принципы: ограничение незавершенной работы (WIP), непрерывный поток, приоритизация задач, минимизация времени выполнения.
